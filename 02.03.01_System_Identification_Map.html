System Identification — KPI → primary system

Tap each KPI signal to reveal its primary system and most likely failure mechanism.

This is a causal hypothesis, not a categorization. The system identification step produces a specific, testable claim: this KPI is moving because this system is failing to do its job.
One primary system per diagnosis. If a second system is contributing, note it as secondary. Multiple co-primaries indicate an incomplete KPI signal — return to Step 1.
